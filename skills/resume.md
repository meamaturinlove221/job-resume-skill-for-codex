## 执行步骤

### Step 1: 读取技能档案
从 `~/.claude/projects/-YOUR_PATH/memory/user_profile.md` 获取完整档案

### Step 2: 获取JD
- 如果用户提供链接，使用 webReader 获取 JD 内容
- 提取并分级关键词：
  - **P1 必须项**：硬性年限要求、必备技能、学历要求
  - **P2 重要项**：核心能力、经验要求
  - **P3 加分项**：软技能、行业知识、加分技能

### Step 3: 匹配分析
将 JD 要求逐条与技能档案匹配：
- 直接匹配的经历和技能
- 可迁移的跨领域能力
- 需要弱化或省略的不相关经历
- 独特优势点

### Step 4: 生成简历

**结构（1页）**：
1. **姓名 + 联系方式**：居中，一行
2. **个人简介**（2-3行）：定位 + 核心技能 + 独特价值
3. **核心技能**：按JD要求分类，使用JD原文术语
4. **工作经历**：倒序，每段突出与JD匹配的成就
5. **教育背景**：学校+学位+时间+GPA
6. **技能/证书**：与JD相关的证书和技能

**撰写原则**：
- 每条经历用 **动词 + 内容 + 数据结果** 格式（如"策划XX活动，带动数据增长187%"）
- 优先排列与JD最匹配的工作内容
- 使用JD中的关键词（ATS优化）
- 所有内容必须真实，不编造经历
- 数据驱动，保留所有量化成果

### Step 5: 输出
- 生成 Markdown 格式简历
- 保存到 `~/Desktop/resumes/{{公司名}}_{{岗位名}}.md`
- 输出匹配度评估 + 简历修改建议 + 面试准备要点

### Step 6: 策略建议
生成简历后额外提供：
- **优势分析**：候选人的核心竞争力
- **差距分析**：未完全满足的要求及弥补建议
- **面试准备**：关键话术、需要准备的故事
- **求职信切入点**：2-3个开篇角度


## 关键节点检查（强制）

**每完成一个步骤，把中间结果发给用户，等用户确认后才继续下一步。禁止不经确认跑完整个流程。**
